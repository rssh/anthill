## reflect.KB operations as eval builtins (in progress 2026-04-23)

`KB.execute` was the only `anthill.reflect.KB.*` operation wired as an eval builtin. The remaining operations declared in `stdlib/anthill/reflect/reflect.anthill` were accessible only through the host-Rust `anthill_stl::reflect::KB` trait (`bridge.rs`), not callable from anthill scripts. Starting here.

**Landed (this iteration):** `anthill-stl/src/reflect/builtins.rs::register_reflect_builtins` registers 23 new eval builtins:

- `anthill.reflect.KB.sort_template`, `KB.sorts`, `KB.operations`, `KB.constructors`, `KB.fields`, `KB.rules`, `KB.descriptions`
- `anthill.reflect.KB.reify`, `anthill.reflect.KB.reflect` (roundtrip verified)
- `anthill.reflect.KB.nonvar`, `anthill.reflect.KB.ground` (eval-time, no DELAY)
- `anthill.reflect.qualified_name`, `anthill.reflect.short_name`, `anthill.reflect.lookup_symbol`, `anthill.reflect.scope`, `anthill.reflect.kind`
- `anthill.reflect.sort_as_term`, `anthill.reflect.term_as_sort`, `anthill.reflect.can_be_sort`
- `anthill.reflect.field_access`, `anthill.reflect.resolve_sort_instantiation_param`
- `anthill.reflect.Substitution.apply`, `anthill.reflect.Substitution.compose`

Plus a new first-class `Value::Substitution(SubstHandle)` variant and `SubstArena` with arena-refcount semantics (mirroring `StreamHandle`). `LogicalStream.splitFirst` now yields `Value::Substitution` instead of the placeholder `Value::Unit`; anthill code can hold, apply, and compose substitutions directly.

Each wraps the existing introspection logic (mirroring `bridge.rs`) and returns `Value::Entity` / `Value::Str` / `Value::Term` results matching the sort shapes in `reflect.anthill`. Scripts now call these through the eval path; host Rust callers keep using `bridge.rs`.

Supporting core additions: `KnowledgeBase::kind_of(sym)` and `KnowledgeBase::scope_of(sym)`, `SymbolTable::iter_defs()`. `register_if_present` + `expect_args` exported from `anthill-core::eval::builtins` for downstream reuse. Symbols resolved once into a cached `ReflectSyms` struct at registration time (no per-call string lookups on the hot path).

Small core change: `Interpreter::call` checks the builtin registry before demanding an operation body, so builtin-only operations are invocable via the top-level API.

**Tests:** `cargo test -p anthill-stl` — 16/16 passing (5 existing bridge tests + 11 new reflect-builtin tests). Full workspace tests green. New tests include `split_first_yields_substitution_values`, `substitution_apply_rewrites_term`, `subst_arena_reclaims_on_drop`.

**Remaining (deferred — need substrate WIs):**

- `anthill.reflect.not(query)` — eval-side negation wrapper. Requires invoking the resolver from inside eval (`kb.resolve(&[goal], &config)` and checking for empty result). Open design question around what "not" means outside SLD context; punt to a dedicated WI.

**Split to follow-up WIs:**

- Mutation builtins (`KB.assert`, `KB.add_guard`) — proposal 023 territory.
- Occurrence ops (`occurrence_term`, `occurrence_span`, `occurrence_owner`, `sub_occurrences`, `occurrences_of`) — proposal 022 territory.
- `instantiation_query` — needs semantics design (bridge.rs stubs it).
- Consolidate bridge.rs helpers with builtins.rs introspection (dedup).

## Verdict

Core execute API delivered. Reflect-builtin layer is in progress; first batch landed and tested. Not ready to deliver until the remaining ops in scope ship.
